Tarea: extraer UNA fecha/hora de notificacion desde una nota en espanol.

Salida obligatoria (sin excepciones):
- SOLO una linea con:
  1) un datetime ISO-8601 con offset (ej: 2026-04-20T09:00:00+02:00)
  2) o EXACTAMENTE: NONE
- No anadas texto, comillas, markdown ni explicaciones.

Contexto temporal:
- NOW (zona Europe/Madrid): {{NOW_ISO}}
- Interpreta expresiones relativas respecto a NOW: hoy, manana, pasado manana, esta tarde, esta noche, el lunes, la semana que viene, etc.
- Debes razonar con calendario real: si NOW es viernes y la nota dice "el martes que viene", devuelve el proximo martes posterior a NOW.

Reglas de interpretacion:
1) Si hay fecha y hora explicitas, usalas.
2) Si hay fecha pero no hora, usa 09:00:00.
3) Si hay hora pero no fecha:
   - usa hoy si aun no ha pasado esa hora;
   - si ya paso, usa manana.
4) Si hay varias fechas, elige la mas probable para "recordatorio" (normalmente la primera instruccion temporal clara).
5) Si aparece dia y mes sin ano (ej: "20 de abril"), usa el proximo ano/fecha futura valida respecto a NOW.
6) Devuelve NONE solo si NO hay ninguna referencia temporal razonable.
7) Para dias de la semana:
   - "el martes" => el proximo martes futuro respecto a NOW.
   - "el martes que viene" => tambien el proximo martes futuro (no inventes +14 dias salvo que el texto lo diga explicitamente).
   - "este martes" => si aun no paso, ese martes; si ya paso, el proximo martes.
8) Para tramos del dia sin hora exacta:
   - manana -> 09:00:00
   - mediodia -> 12:00:00
   - tarde -> 17:00:00
   - noche -> 21:00:00
9) Si hay ambiguedad leve pero existe una interpretacion temporal razonable, devuelve fecha/hora; usa NONE solo cuando no haya ninguna.

Ejemplos de conversion (formato):
- "avisame manana a las 8" -> YYYY-MM-DDT08:00:00+01:00 (o +02:00 segun DST)
- "recordarme el 20 de abril" -> YYYY-04-20T09:00:00+02:00
- "llamar a mama hoy 19:30" -> YYYY-MM-DDT19:30:00+01:00 (o +02:00 segun DST)
- "el martes que viene revisar facturas" -> YYYY-MM-DDT09:00:00+01:00 (o +02:00 segun DST)
